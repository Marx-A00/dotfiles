#+title: Monitor Setup & Switching Plan

Plan for switching monitors between MrX (MacBook M4 Max) and VENGEANCE
(gaming PC), plus the upgrade path. Based on inventory as of 2026-07.
Hardware details: ~/roaming/notes/mr-x-inventory-mdox.org

* TL;DR

- No KVM box needed. Each S2719DGF has 3 inputs (DP 1.2, HDMI 2.0,
  HDMI 1.4) — plug BOTH computers into BOTH monitors and flip the
  active input over DDC/CI. Fully scriptable from both machines.
- Phase 1 (~$0-25): dual-input wiring + DDC scripts + cheap USB switch
  for keyboard/mic. One-hotkey handoff each direction.
- Phase 2 (~$650-900): one 4K 240Hz QD-OLED with built-in KVM + USB-C
  as the centerpiece. Fixes the two real weaknesses: non-Retina text on
  macOS, and the RTX 5080 being bottlenecked by a 2018 QHD panel.
- Nothing bought in Phase 1 is wasted by Phase 2.

* The key fact

The Dell S2719DGF has three inputs:
- DP 1.2 :: up to 155Hz
- HDMI 2.0 :: up to 144Hz
- HDMI 1.4 :: up to 60Hz

Monitor input selection is controllable over DDC/CI (VCP code 60), so
the "KVM" is just software sending a command over the video cable.

* Phase 1: Best setup with current gear (~$0-25)

** Current wiring (as of 2026-07-17)

- MrX -> iVANKY dock -> HDMI to displays 2 and 3
- VENGEANCE -> DP to displays 3 and 4
- Display 4 "flex" input: work laptop via USB hub, or HDMI manually
  swapped over to MrX when all displays wanted (the annoying part)
- Display 2 not connected to VENGEANCE (fine, not a goal)

** Target wiring — every cable permanent, zero swapping

Key: each S2719DGF has THREE inputs (DP 1.2 / HDMI 2.0 / HDMI 1.4),
so display 4 has a seat for all three machines at once.

- Display 2 (portrait) :: HDMI <- dock (as-is)
- Display 3 (center) ::
  - DP <- VENGEANCE (as-is)
  - HDMI 2.0 <- dock (as-is) — already fully wired
- Display 4 (right) ::
  - DP <- VENGEANCE (as-is)
  - HDMI 2.0 <- dock (THE ONE NEW CABLE)
  - HDMI 1.4 <- permanent parking spot for work laptop's USB hub
    (1440p@60Hz cap on the 1.4 port — fine for work)

Dock-port caveat: if both FusionDock HDMI outs are taken by displays
2+3, run display 4 off a TB4 downstream port with a USB-C->HDMI cable
(~$12) instead.

** Modes (all just input flips, no cables touched)

The "always >= 2 monitors" rule holds on BOTH machines simultaneously:

- Mac mode :: 2, 3, 4 on HDMI -> three externals + laptop screen
- Game mode :: 3 + 4 flip to DP -> PC gets its two 155Hz panels;
  Mac keeps portrait + built-in (still 2 screens)
- Work mode :: only 4 flips to HDMI 1.4 -> work laptop gets a
  monitor; Mac keeps 2 + 3 + built-in

Portrait's job in this layout: Slack/music/htop parking lot + the
insurance policy that guarantees the Mac a second screen in game mode.

** Switching: Mac side

BetterDisplay.app is already installed. Its CLI needs the Pro license
(~$20); the free alternative:

#+begin_src sh
brew install m1ddc
#+end_src

LIVE as of 2026-07-17. =macos/scripts/monitor-mode.sh= (m1ddc,
displays pinned by UUID) supports presets, per-display assignment,
toggles, and status:

#+begin_src sh
monitor-mode.sh game            # center+right -> VENGEANCE
monitor-mode.sh mac             # center+right -> MrX
monitor-mode.sh work            # right -> work laptop
monitor-mode.sh center mac|pc   # mix & match per display
monitor-mode.sh right  mac|pc|work
monitor-mode.sh toggle center|right   # flip Mac <-> PC
monitor-mode.sh status
#+end_src

skhdrc bindings:
#+begin_src sh
ctrl + alt - g : game       # gaming: both mains -> VENGEANCE
ctrl + alt - m : mac        # everything home
ctrl + alt - w : work       # right -> work laptop
ctrl + alt - 3 : toggle center   # numbers match yabai display idx
ctrl + alt - 4 : toggle right    # e.g. movie-on-PC while Mac-working
#+end_src

Creator Micro macro pad (VIA layer 2; F13-F16 are taken by the
agent-shell permission bindings). GOTCHA: macOS ignores F20+ from
hardware keyboards — usable range is F13-F19 only. The 4th key sends
LSFT(KC_F19) via VIA's "Any" keycode (modifier+F17-F19 combos = 9
more free dedicated slots for future pad keys):
#+begin_src sh
f17 : game            f18 : mac
f19 : toggle center   LSFT(KC_F19) : toggle right
LSFT(KC_F17) : work (right -> work laptop)
#+end_src
Keep the Micro plugged into the MAC side (dock) permanently — the Mac
stays alive and DDC-capable in every mode, so the pad can always pull
displays back. It never needs to switch machines.

KEY FINDINGS (verified by live round-trip tests):
- DDC WRITES reach both monitors from the Mac at ALL times, even while
  a monitor displays another input. The Mac alone hands off AND pulls
  back — no PC-side tooling needed for input switching.
- DDC READS only work on center (direct dock HDMI). The Anker 310 on
  right not only fails reads — it sometimes returns GARBAGE with exit
  0. Right's state therefore comes from the state file ONLY
  (~/.local/state/monitor-mode/); never trust a read from it.
- Cable-vs-adapter is NOT the fix for reads: DDC passthrough depends
  on the converter chip inside, not the form factor. Only swap if a
  specific model is community-confirmed DDC-transparent.

** Windows topology sync (no phantom displays)

Problem: Windows keeps a monitor in its extended desktop even while
that monitor shows the Mac, so apps/games can launch invisibly.
Fix: after every input flip, =sync_windows()= in monitor-mode.sh
fire-and-forgets an SSH trigger of a scheduled task on VENGEANCE:
- mon-extend :: both 3+4 on Windows (or nothing on Windows — idle)
- mon-only4 :: only 4 on Windows (movie mode) — disables 3
- mon-only3 :: only 3 on Windows — disables 4

VENGEANCE-side plumbing (all under =C:\Tools\MultiMonitorTool\=):
- MultiMonitorTool.exe (NirSoft) :: /disable <serial> WORKS on Win11;
  /enable, /SetMonitors are BROKEN (silently no-op) — as are
  DisplaySwitch.exe /extend from a task.
- extend.ps1 :: the only thing that re-enables: direct
  SetDisplayConfig(0,0,0,0, SDC_APPLY|SDC_TOPOLOGY_EXTEND) P/Invoke.
- Monitor serials: C3GZBY2 = display 3 (center), D8TYBY2 = display 4
  (right). Windows \\.\DISPLAYx numbers shift; serials don't.
- Scheduled tasks run interactive-session-only (created over SSH
  without /ru), because display APIs are dead in detached SSH
  sessions. Helper tasks: mmt-dump (CSV state dump), mmt-exec (runs
  whatever is in cmd.txt — line must be CRLF-FREE, a trailing \r
  silently corrupts the last argument).

DONE 2026-07-17: both monitors at 155Hz (required enabling the
FACTORY OVERCLOCK in each monitor's OSD — Menu -> Display ->
Overclock; the S2719DGF is natively 144Hz, 155 is OC-only and
DP-only) + G-Sync Compatible enabled in NVIDIA Control Panel.
Verified over SSH: both panels 2560x1440 @ 155Hz.

Display 4 negotiated full 2560x1440@144Hz through the Anker 310
(dock TB4 port). All four displays at max spec.

Common Dell VCP 60 input values:
- DP :: 15
- HDMI 1 :: 17
- HDMI 2 :: 18

CONFIRMED 2026-07-17 via BetterDisplay DDC read (=BetterDisplay get
-namelike=S2719 -ddc -vcp=inputSelect -value= -> 18 while the Mac
drives it at 144Hz): the HDMI 2.0 port = 18 ("HDMI 2"), so HDMI 1.4 =
17 and DP = 15. The mode bindings above use the right values.

Current-wiring audit (same date): display 3 on HDMI 2.0 @144Hz ✓,
display 2 @100Hz (its max) ✓ — Mac side is plugged in optimally.

Note: BetterDisplay CLI works for DDC reads with the app running
(=open -ga BetterDisplay=) — may be an alternative to m1ddc for the
bindings if Pro license is active.

** Switching: Windows side

NirSoft ControlMyMonitor, bound to a hotkey with AutoHotkey:

#+begin_src sh
# hand back to the Mac (HDMI = 17)
ControlMyMonitor.exe /SetValue <monitor> 60 17
#+end_src

One hotkey on each side = round trip.

** Peripherals — dedicated seats, (almost) nothing switches

Same philosophy as the monitors: multi-device gear gets assigned, not
swapped.

*** Keyboards
- ErgoDox Hot Dox V2 :: wired to dock = Mac daily driver
- Keychron V6 :: wired to VENGEANCE permanently (QMK, full-size)
- Keychron B1 Pro :: work laptop board (tri-mode: BT x3 + 2.4GHz +
  wired). Pair BT2 = Mac, BT3 = VENGEANCE -> fn+1/2/3 hops machines.
  The floater.
- YUNZII KC68 :: bench/backup (Gateron Blues = solo-room board)
- Magic Keyboard :: bench, or couch/homelab keyboard

*** Mice
- Logitech G502 X Plus :: VENGEANCE forever. Lightspeed dongle ONLY —
  no Bluetooth — so the dongle never leaves the PC. Gaming mouse anyway.
- Keychron M3 Mini 4K :: Mac daily (dongle in dock), BT slots -> work
  laptop etc. The floater.
- GravaStar Mercury M2 :: work laptop dedicated, or bench

*** Speakers (zero switching, ever)
Audioengine A2+ mixes all active inputs simultaneously — no selector:
- USB <- dock/Mac (best quality, internal DAC)
- RCA/3.5mm <- VENGEANCE (wired = zero latency for gaming; never BT)
- Bluetooth <- stays free for phone
- S8 sub hangs off the A2+ as-is

*** Mic (Shure MV7+)
- USB-C -> VENGEANCE (Discord lives there)
- If also wanted for Mac calls: cheap USB 3.0 switch (~$20, UGREEN)
  between dock and PC — the ONLY thing left that needs a switch
- Optional automation: fire DDC switch on USB connect events
  - Example: https://gist.github.com/balantrapu87/269c380e23a29a9f78966e9a950e7418
  - Pattern writeup: https://joostrijneveld.nl/posts/2024-06-04-ddc-input-switching/

*** Net result
Machine swap ritual = monitor-mode hotkey + grab the other keyboard.
Every machine always has its own kb+mouse; audio just works.

** Physical KVM verdict: SKIP

- Level1Techs dual-monitor DP 1.4 KVM (gold standard): $520 and sold out
- Cheap TESmart-tier KVMs are flaky at high refresh
- KVM boxes only make sense when monitors lack spare inputs — mine don't

* Phase 2: Best possible setup (new gear)

** The two real weaknesses of current gear

1. macOS on 27" 1440p is non-Retina (108ppi, no HiDPI) — text in Emacs
   is noticeably softer than the MacBook's screen. Biggest QoL gap for
   staring-at-text-all-day work.
2. RTX 5080 is wildly bottlenecked by a 2018 QHD panel — that GPU is
   built for 4K 240Hz.

One monitor fixes both.

** The move: 4K 240Hz QD-OLED with built-in KVM + USB-C

Candidates (2026 prices):
- Gigabyte MO27U2 :: 27", 4K 240Hz QD-OLED, USB-C 90W PD, built-in
  KVM, ~$650. The value pick.
- MSI MPG 321URX :: 32", same panel class, ~$830. Widely considered
  the best 32" OLED with KVM.
- ASUS ROG PG32UCDM / PG32UCDM3 :: ~$900. Premium version of same idea.

Why built-in KVM is the future-proof answer:
- Mac connects with ONE USB-C cable (video + USB + 90W charging)
- PC connects via DP
- Monitor's internal USB hub follows the active input automatically
- Keyboard + mic plug into the monitor; switching = one button
  (still DDC-scriptable too)

macOS rendering:
- 4K @ 27" = 163ppi -> HiDPI "looks like 1440p", MacBook-screen crisp
- 4K @ 32" = 140ppi -> "looks like 2560x1440" comfortable

** End state layout

- Center: 4K OLED (shared Mac/PC)
- Flank: one S2719DGF landscape
- Side: S2725HS portrait (unchanged)
- Spare S2719DGF: second monitor for VENGEANCE, homelab corner, or
  sell (~$120-150 used)

** OLED burn-in caveat

Real but managed in 2026 — panels run compensation cycles, and mixed
gaming/work use actually helps. Static UI elements (menu bar,
sketchybar, modeline) are the risk. If it spooks:
- Boring-but-bulletproof alternative: 27" 4K IPS with built-in KVM
  (Gigabyte M27U class, ~$400) — same Retina + KVM benefits, zero
  burn-in anxiety, less gaming wow.
